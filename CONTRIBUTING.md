# Contributing to Lattice

First off, thank you for considering contributing to Lattice! It's people like you that make Lattice a great tool for the AI agent community.

## How Can I Contribute?

### Reporting Bugs
- Use the [GitHub Issue Tracker](https://github.com/YuvalHir/Lattice/issues).
- Describe the bug and provide steps to reproduce it.
- Mention your OS and Lattice version.

### Suggesting Enhancements
- Open a GitHub issue with the tag `enhancement`.
- Describe why this feature would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue a pull request!

## Development Setup

See the [README](README.md#for-developers-build-from-source) for instructions on setting up your local environment.

### Project Structure
- `src/`: SolidJS frontend.
- `src-tauri/`: Rust backend.
- `docs/`: Architectural documentation.

## Style Guide
- **Rust**: Use `cargo fmt` and `clippy`.
- **Frontend**: Use Prettier and follow the existing TypeScript patterns.
- **Commit Messages**: We prefer [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/).

## Community
If you have questions, feel free to open a discussion on GitHub.
